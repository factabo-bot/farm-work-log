# セットアップ手順

段階ごとに進める。**ステップAは今すぐ・設定なしで動く。**

---

## ステップA: ブラウザで触ってみる（設定不要・今すぐ）

1. `frontend/index.html` をダブルクリックしてブラウザで開く
2. 「お試しモード」と表示され、記録はこの端末のブラウザ内（localStorage）にだけ保存される
3. 棟 → マスをタップ → 作業を選択 → 記録する、の流れを確認
4. 記録後にもう一度開くと、記録済みのマスがグレーになっていることを確認

スマホで試すには: PCと同じWi-FiにあるスマホからはそのままはアクセスできないのでステップCのGitHub Pages公開後に確認するのが簡単。

UIの調整（列数・棟名・作業の種類）は `frontend/masters.js` を書き換えるだけ。

---

## ステップB: スプレッドシート＋GAS（記録の保存先を作る）

1. [Google スプレッドシート](https://sheets.google.com)で新規シート作成（名前: 例「作業記録DB」）
2. メニュー「拡張機能」→「Apps Script」
3. エディタが開いたら、`gas/Code.gs` の中身を全部貼り付けて保存
4. エディタ上部の関数選択で `setup` を選んで「実行」（初回は権限の承認ダイアログが出る → 許可）
   - スプレッドシートに「記録」「スタッフ」シートが自動作成される
5. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
6. 発行されたURL（`https://script.google.com/macros/s/…/exec`）をコピー
7. `frontend/config.js` の `GAS_URL` に貼り付ける

> コードを修正して再デプロイするときは「デプロイを管理」→ 鉛筆アイコン → バージョン「新バージョン」→ デプロイ。URLは変わらない。

---

## ステップC: ホスティング（スマホから開けるURLにする）

GitHub Pagesを使う（無料・URL固定）。

1. GitHubに新しいリポジトリを作る（例: `farm-work-log`、Public）
2. このプロジェクトをpushする:
   ```
   cd C:\Users\facta\projects\farm-work-log
   git init
   git add .
   git commit -m "v0: 作業記録画面"
   git remote add origin https://github.com/<ユーザー名>/farm-work-log.git
   git push -u origin main
   ```
3. リポジトリの Settings → Pages → Branch: main / フォルダ: `/ (root)` → Save
4. 数分後 `https://<ユーザー名>.github.io/farm-work-log/frontend/` で開ける
5. スマホのブラウザでこのURLを開いて動作確認（この時点ではまだお試しモード or GAS保存）

> 注意: Publicリポジトリなのでコードは公開される。`config.js` に入れるのはGASのURLとLIFF IDのみで、これらは秘密情報ではない（GAS側は「URLを知っていれば書き込める」状態なので、運用開始後に問題があればトークン照合を足す）。

---

## ステップD: LINE連携（LIFF化）

1. [LINE Developersコンソール](https://developers.line.biz/console/)に個人のLINEアカウントでログイン
2. プロバイダーを新規作成（名前: 例「farm-tools」）
3. 「LINEログイン」チャネルを作成
   - チャネル名: 例「作業記録」
   - アプリタイプ: ウェブアプリ
4. 作成したチャネルの「LIFF」タブ →「追加」
   - LIFFアプリ名: 作業記録
   - サイズ: **Full**
   - エンドポイントURL: ステップCのURL（`https://…/frontend/`）
   - Scope: **profile** にチェック
5. 発行された **LIFF ID**（例 `1234567890-abcdefgh`）を `frontend/config.js` の `LIFF_ID` に設定し、push（Pagesに反映）
6. LIFF URL（`https://liff.line.me/<LIFF ID>`）を自分のLINEの「Keepメモ」に送ってタップ → LINE内で開き、自分の名前が表示されれば成功

---

## ステップE: テストグループで運用テスト

1. 自分＋家族（またはサブ端末）でLINEグループを作る
2. グループの「ノート」にLIFF URLを投稿して固定
3. 数日間、自分の研修作業を実際に記録してみる
4. スプレッドシートに「1マス1行」でデータが溜まることを確認

---

- 「次はこれかも」提案機能

- マスタのシート化（現状は `masters.js` を直接編集）
